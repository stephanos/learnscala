Erste Schritte

Ein kleiner Schritt für einen Entwickler ...

Erste Eingabe in die REPL

@comp.snippet { |"Hello World" }
@comp.console { |res0: java.lang.String = Hello World }
  • res0 ist der Name der Variable
    (fortlaufend nummeriert)
  • java.lang.String ist Typ der Variable
    (automatisch ermittelt)
  • Hello World ist Wert der Variable
    (via toString())

Variablen

@comp.snippet { |val hello: String = "Hello" }
  • val beginnt Variable
  • hello ist der Name
  • : trennt Name und Typ
  • String ist der Typ
  • = weist Wert zu
  • "Hello" ist der Wert
@comp.console { |hello: java.lang.String = Hello }
@comp.shout("info-circle") { val leitet eine Variable ein,
der Typ wird hinter dem
Namen angegeben. }

Variablen

@comp.snippet { |val hello: String = "Hello" | |hello = "Goodbye" }
@comp.console { |hello: String = Hello | |:8: error: reassignment to val | hello = "Goodbye" }
@comp.shout("info-circle") { val kann nicht überschrieben werden, sie ist unveränderlich (engl. "immutable"). }

Variablen

@comp.snippet { |val hello: Int = "Hello" }
@comp.console { |:7: error: type mismatch; | found : java.lang.String("Hello") | required: Int | val hello: Int = "Hello" | ^ }
@comp.shout("info-circle") { Der Compiler steht sicher, dass Variablentyp und -wert zusammenpassen. }

Variablen

@comp.snippet { |val hello1: String = "Hello" |val hello2 = "Hello" } @comp.console { |hello1: java.lang.String = Hello |hello2: java.lang.String = Hello }
@comp.snippet { |val meaningOfLife = 42 |val pi = 3.14f }
@comp.console { | } @comp.shout("info-circle") { Der Variablentyp kann weggelassen werden, der Compiler leitet ihn ab. }

Variablen mit var

@comp.snippet { |var hello = "Hello World" |hello = "Hello Mars" } @comp.console { |hello: java.lang.String = Hello World |hello: java.lang.String = Hello Mars }
  • Wert ist veränderlich ("mutable")
    • entspricht Standardverhalten in Java
    • problematisch bei Zugriff von mehreren Threads
  • Tipp: nur in Ausnahmen verwenden (z.B. Performance)

Optionale Typenangabe

Optionale Semikolons

@comp.snippet { |val hello1 = "Hello" |val hello2 = "Hello"; } @comp.console { |hello1: java.lang.String = Hello |hello2: java.lang.String = Hello }
@comp.snippet { |val h1="Hi"; val h2="Hi" } @comp.console { |hello: java.lang.String = Hello |hello2: java.lang.String = Hello }
  • Tipp: Semikolons weglassen
  • Ausnahme: mehrere Anweisungen in einer Zeile

Methodendefinition

@comp.snippet { |def hello(): String = { | return "Hello World" |} } @comp.console { |hello: String }
  • def startet die Definition
  • hello ist der Name der Method
  • () beinhaltet die Liste der Parameter
  • String ist der Rückgabetyp der Methode
  • { .. } enthält Implementierung der Methode
  • Regel: Jede Methode hat einen Rückgabewert.
Difference between method and function in Scala

Methodendefinition: Typ Unit

@comp.snippet { |def hello() { | println("Hello World") |} } @comp.console { |hello: Unit }
  • Definition ohne =
  • entspricht void aus Java
Difference between method and function in Scala

Methodendefinition: Optionales

@comp.snippet { |def hello(): String = | return "Hello World" }
  • ohne geschweifte Klammern
    Wann? Nur eine Zeile
@comp.snippet { |def hello(): String = | "Hello World" }
  • ohne return
    Wann? Immer, Rückgabe ist Ergebnis des letzten Ausdrucks
@comp.snippet { |def hello: String = | "Hello World" }
  • ohne Parameterliste
    Wann? Keine Parameter
@comp.snippet { |def hello = | "Hello World" }
  • ohne Rückgabetyp
    Wann? Nicht rekursiv oder überladen
@comp.snippet { |def hello = "Hello World" }
  • ohne Zeilenumbruch

Methodendefinition: Parameter

@comp.snippet { |def add(x: Int, y: Int) = x + y } @comp.console { |add: (x: Int, y: Int)Int }
  • erwartet 2 Parameter vom Typ Int
  • Trennung durch Komma
  • Typen müssen definiert werden

Vergleich von def, val und var

@comp.snippet { |val rand = util.Random.nextInt(10) |println(rand) |println(rand) } @comp.console { |rand: Int = 6 |6 |6 }
  • ⇨ var / val wird einmal ausgeführt
@comp.snippet { |def rand = util.Random.nextInt(10) |println(rand) |println(rand) } @comp.console { |rand: Int |3 |9 }
  • ⇨ def wird einmal bei jedem Aufruf ausgeführt

Zusammenfassung

  • bevorzugt unveränderliche Variablen via val
  • Datentypdefinition stets am Ende (z.B. def hello(name: String): String)
  • optionale Semikolons, Datentypen und Klammern
