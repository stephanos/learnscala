Erste Schritte

Ein kleiner Schritt für einen Entwickler ...

Erste Eingabe in die REPL

@comp.snippet { |"Hello World" } @comp.console { |res0: java.lang.String = Hello World }
  • res0
    Name der Variable, fortlaufend nummeriert
  • java.lang.String
    Typ der Variable, automatisch ermittelt
  • Hello World
    Wert der Variable (via toString())

Werzuweisung mit val

@comp.snippet { |val hello = "Hello World" } @comp.console { |hello: java.lang.String = Hello World }
  • Wert ist unveränderlich ("immutable")
    • keine Seiteneffekte und dadurch weniger Komplexität
    • leichter zu parallelisieren und zu testen
  • entspricht final String hello in Java
  • Regel: bevorzugte Verwendung von val
  • Außname: bei sehr großen Daten

Werzuweisung mit var

@comp.snippet { |var hello = "Hello World" |hello = "Hello Mars" } @comp.console { |hello: java.lang.String = Hello World |hello: java.lang.String = Hello Mars }
  • Wert ist veränderlich ("mutable")
  • entspricht Standardverhalten in Java
  • Regel: zu vermeiden wenn möglich

Optionale Typenangabe

@comp.snippet { |val hello: String = "Hello" } @comp.console { |hello: java.lang.String = Hello }
  • Regel: Typen für temporäre Variablen werden weggelassen
  • Ausnahme: Dokumentationszwecke oder falsche Erkennung

@comp.snippet { |val hello: Int = "Hello" } @comp.console { |:7: error: type mismatch; | found : java.lang.String("Hello") | required: Int | val hello: Int = "Hello" | ^ }
  • Fehlererkennung durch den Compiler dank Typensicherheit

Optionale Semikolons

@comp.snippet { |val hello = "Hello" |val hello2 = "Hello"; } @comp.console { |hello: java.lang.String = Hello |hello2: java.lang.String = Hello }
  • Regel: Semikolons werden weglassen
@comp.snippet { |val h1="Hi"; val h2="Hi" } @comp.console { |hello: java.lang.String = Hello |hello2: java.lang.String = Hello }
  • Außnahme: mehrere Anweisungen in einer Zeile
