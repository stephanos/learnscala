Werkzeuge

Homo scalacus

@overview

scala-compiler.jar

  • rund 11 MB groß
  • benötigt zum
    Kompilieren
  • erstellt Java-Bytecode
    aus Scala- und Java-Dateien

scala-library.jar

  • rund 8.6 MB groß
  • benötigt zum
    Kompilieren und Ausführen
  • enthält Scala-Klassen
    und -Sprachelemente
    (wie Collections, Annotationen etc.)
@overview
@comp.headbody() { > scalac } {
  • Konsolenanwendung: kompiliert Scala-Code zu Java-Bytecode
    @comp.outcode(fragment = true) { |$ scalac HelloWorld.scala }
  • braucht typischerweise etwas länger als der Java-Compiler javac
}
@comp.headbody() { > fsc } {
  • fast scala compiler
  • Konsolenanwendung: kompiliert Scala-Code mittels Hintergrundprozess
  • kompiliert schneller als scalac: Startzeit fällt weg plus Caching
    @comp.outcode() { |$ fsc HelloWorld.scala // erster Aufruf, Prozess wird gestartet und kompiliert | |$ fsc HelloWorld.scala // zweiter Aufruf, Prozess kompiliert direkt }
}
@comp.headbody() { > scala } {
  • Konsolenanwendung: führt Scala-Programme aus
    1. kompilierte Programme
      @comp.outcode() { |$ scala HelloWorld | |$ Hello World }
      alternativ kann natürlich auch java verwendet werden:
      @comp.outcode() { |$ java -classpath lib/scala-library.jar HelloWorld | |$ Hello World }
    2. interaktive Programme
      @comp.outcode() { |$ scala | |scala> 1 + 2 |res0: Int = 3 }
}
@overview
@comp.headbody() { Maven } {
  • offizielles Scala-Plugin
  • kompiliert src/main/scala und src/test/scala
  • nur leider zu langsam,
    lieber SBT verwenden
@comp.outcode() { |<plugin> | <groupId>org.scala-tools</groupId> | <artifactId>maven-scala-plugin</artifactId> | | <executions> | <execution> | <goals> | <goal>compile</goal> | <goal>testCompile</goal> | </goals> | </execution> | </executions> | | <configuration> | <scalaVersion>${scala.version}</scalaVersion> | </configuration> |</plugin> }
}
@comp.headbody() { SBT } {
  • Simple Build Tool, aktuelle stabile Version 0.11.3 (nicht abschrecken lassen!)
    • ist in Scala geschrieben
    • kompatibel mit Maven-Repositories
    • Projektdefinition in Scala, mit Hilfe einer speziellen DSL
    • läuft permanent als Prozess im Hintergrund: schnelles Kompilieren
    • viele Scala-Bibliotheken, Scala selbst und Typesafe setzen darauf
}

@comp.outcode() { |$ sbt |> compile } }
