Methoden

und Ihre Definition

@comp.headbody() { Methoden: Definition } {
@comp.snippet() { |def hello(): String = { | return "Hello World" |} } @comp.console() { |hello: String }
  • def startet die Definition
  • hello ist der Name der Methode
  • () beinhaltet die Liste der Parameter
  • String ist der Rückgabetyp
  • { .. } enthält Implementierung
@comp.infobox() { def leitet eine Methode ein - diese hat immer einen Rückgabetyp. } }
Difference between method and function in Scala
@comp.headbody() { Methoden: Unit } {
@comp.snippet() { |def hello() = { | println("Hello World") |} }
@comp.console() { |hello: Unit }
@comp.snippet() { |def hello() { | println("Hello World") |} } @comp.console() { |hello: Unit }
@comp.infobox() { Hat eine Methode keinen Rückgabewert, hat sie den Typ Unit. } }
@comp.headbody() { Methoden: Optionales } {
@comp.snippet(id = "hello") { |def hello(): String = | return "Hello World" }
  • ohne geschweifte Klammern
    Wann? Nur eine Zeile
@comp.snippet() { |def hello(): String = | "Hello World" }
  • ohne return
    Wann? Immer, Rückgabe ist Ergebnis des letzten Ausdrucks
@comp.snippet() { |def hello: String = | "Hello World" }
  • ohne Parameterliste
    Wann? Keine Parameter
@comp.snippet() { |def hello = | "Hello World" }
  • ohne Rückgabetyp
    Wann? Nicht rekursiv oder überladen
@comp.snippet() { |def hello = "Hello World" }
  • ohne Zeilenumbruch
}
